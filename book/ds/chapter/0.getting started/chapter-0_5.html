Chapter 0.5

Data Science Workflow

11 min read Beginner Foundation

Every successful data science project follows a structured workflow. Understanding this process helps you navigate from raw data to actionable insights efficiently.

The Complete Workflow

Data science isn't a linear process—it's iterative and cyclical. However, there's a general framework that most projects follow. Think of it as a roadmap rather than a rigid set of rules.

Key Insight: You'll often loop back to earlier steps as you learn more about your data and problem. This is normal and expected in data science.

Step 1: Business Understanding

Before touching any data, you need to understand the business problem you're solving. This is the most critical step—solve the wrong problem perfectly, and you've wasted everyone's time.

Key Questions

What problem are we solving? Why does it matter? What does success look like? Who will use the results?

Stakeholder Alignment

Meet with business owners, understand constraints, define metrics, set expectations.

Define Success

Establish clear, measurable objectives. What accuracy do we need? What's the timeline?

Step 2: Data Collection

Once you understand the problem, identify and gather the data you need. This might come from databases, APIs, files, web scraping, or external sources.

Common Data Sources

  • Internal databases (SQL, NoSQL)
  • APIs and web services
  • CSV/Excel files
  • Web scraping
  • Third-party data providers

Key Considerations

  • Data availability and access
  • Data quality and completeness
  • Privacy and compliance (GDPR, etc.)
  • Volume and storage requirements
  • Cost of data acquisition

Step 3: Data Cleaning

Real-world data is messy. This step often takes 50-80% of project time. You'll handle missing values, fix errors, remove duplicates, and standardize formats.

"Data cleaning is the unglamorous but essential foundation of any successful data science project."

— Data Science Reality
1

Handle Missing Data

Decide whether to remove, impute, or flag missing values.

2

Remove Duplicates

Identify and eliminate redundant records.

3

Fix Errors

Correct typos, standardize formats, validate data types.

4

Handle Outliers

Investigate anomalies—are they errors or valuable insights?

Step 4: Exploratory Data Analysis (EDA)

Now you explore your clean data to understand patterns, distributions, and relationships. This is where you develop intuition about what the data is telling you.

Visualize

Create histograms, scatter plots, box plots, and correlation matrices to see patterns.

Summarize

Calculate statistics: means, medians, standard deviations, percentiles.

Investigate

Look for trends, seasonality, correlations, and unexpected patterns.

EDA Goals: Understand your data's shape, spot quality issues, identify relevant features, and form hypotheses to test.

Step 5: Feature Engineering

Transform raw data into features that better represent the underlying problem. This is often the difference between a mediocre model and a great one.

Common Techniques

  • Creating new features from existing ones
  • Encoding categorical variables
  • Scaling and normalization
  • Binning continuous variables
  • Feature extraction (PCA, embeddings)

Examples

  • Extract day of week from timestamp
  • Calculate customer lifetime value
  • Create interaction terms
  • Aggregate transaction history
  • Text features from descriptions

Step 6: Modeling

Select and train machine learning models (or statistical methods) that fit your problem type. Start simple, then increase complexity if needed.

1

Choose Algorithm

Select based on problem type: regression, classification, clustering, etc.

2

Split Data

Divide into training, validation, and test sets.

3

Train Model

Fit the model to training data and tune hyperparameters.

4

Validate

Test on validation set to prevent overfitting.

Important: Start with simple models (linear regression, logistic regression) before trying complex ones (deep learning). Simple often wins.

Step 7: Evaluation

Assess your model's performance using appropriate metrics. Does it meet the business requirements defined in Step 1?

Choose Metrics

Accuracy, precision, recall, F1-score, RMSE, R-squared—pick what matters for your problem.

Test on Holdout

Use the untouched test set to get unbiased performance estimates.

A/B Testing

For production systems, test your model against the current solution in real conditions.

Step 8: Communication & Deployment

The final step is sharing your findings and putting your solution into production. Even the best model is worthless if stakeholders don't understand or can't use it.

Communication

  • Create clear visualizations
  • Write executive summaries
  • Present findings to stakeholders
  • Document methodology
  • Explain limitations and assumptions

Deployment

  • Package model for production
  • Set up monitoring and alerts
  • Plan for model retraining
  • Document maintenance procedures
  • Ensure scalability and reliability

"A model in a Jupyter notebook has zero business value. Only deployed, monitored, and maintained models create impact."

— Production ML Wisdom

The Iterative Nature of Data Science

Remember: this workflow isn't a one-way street. You'll constantly loop back to earlier steps based on what you learn.

Common Loops

Poor model performance? Go back to feature engineering or data collection. New insights during EDA? Refine your business understanding.

Continuous Improvement

Even after deployment, monitor performance and retrain models as data changes and business needs evolve.

Learning Process

Each iteration teaches you more about the problem, the data, and the domain. Embrace the learning.

Final Wisdom: Perfect is the enemy of good. Ship a working solution, gather feedback, and improve iteratively rather than seeking perfection upfront.
